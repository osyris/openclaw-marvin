## CallVA (callva.io)

### ⚠️ Стратегическая заметка (25.02.2026)
Опасность: CallVA рискует стать "человеком-оркестром" — умеет всё, но непонятно что конкретно делает. Нужно **одно острое основное предложение** для пробития рынка. Остальные юзкейсы — вспомогательные, описываются в доке, но требуют отдельного позиционирования на будущее. Размытое "платформа для всего" не конвертит.

### Позиционирование
**НЕ движок звонков** (как VAPI/Retell/Bland), а **оркестрирование бизнес-целей** через голосовых ассистентов.

### Ключевое отличие: бизнес-ориентированный UI
- Конкуренты: просто списки звонков, оркестрирование агентов
- CallVA: **UI тонко настраивается под конкретную бизнес-специфику**
  - Статусы: scheduling → in progress → complete / failed
  - Результаты зависят от типа звонка
  - Метрики имеющие значение для бизнеса, а не просто "звонок был"
  - Повторные дозвоны, логика перезвонов

### Project Context Framework (фича для AI-работы с проектом)

Каждый проект в CallVA должен иметь структурированный "контекст" — набор документов/полей, которые позволяют AI-модели понимать проект и работать с ним эффективно. Это универсальный шаблон (сначала для CallVA, потом для любых проектов вообще).

**Обязательные блоки:**

**1. Goal (Цель)**
- Что данный проект делает? Одно предложение, чётко.
- Пример: "Напоминаем пациентам о предстоящих визитах за 24ч и подтверждаем запись"

**2. Success Criteria (Когда считаем, что работает хорошо)**
- Конкретные наблюдаемые состояния, а не метрики
- Пример: "Пациент подтвердил визит / перенёс / отказался — ответ зафиксирован"
- Пример: "Агент не перебивает, когда пациент говорит"

**3. Negative Criteria (Чего не делаем / избегаем)**
- Явные запреты и нежелательные сценарии
- Пример: "Агент не звонит повторно если пациент уже ответил"
- Пример: "Не спрашиваем о диагнозе / медицинских деталях"

**4. Success Metrics (Измеримые показатели)**
- Вырастают из Success Criteria
- Пример: % подтверждённых визитов, % no-show после звонка, средняя длина звонка

**Зачем это:**
- AI-модель, работающая с проектом, понимает "точку Б" — куда идём
- Из этой точки Б вырастают все решения: какие поля в звонке, как анализировать результат, какой промпт использовать
- Когда что-то не работает — возвращаемся к критериям и смотрим где расхождение
- Универсально: этот фреймворк применим к любому бизнес-проекту, не только к звонкам

### Целевые аудитории

**1. Бизнесы напрямую (self-serve)**
- Заходит → шаблон → 5 кликов → агент принимает звонки
- Пополняет баланс, списание за минуты и сервисы
- Настройка через разговор с внутренним агентом-аккаунт-менеджером

**2. Технические люди / агентства**
- Строят решения для других бизнесов
- Используют CallVA как дашборд + панель для своих клиентов
- Управление несколькими агентами, права доступа

**3. Партнёры/реселлеры**
- Продают голосовых ассистентов бизнесам
- Настраивают, поддерживают, решают проблемы
- Доход: % с продажи + % с минут

### Архитектура: полностью API-driven

**CallVA = оркестратор бизнес-задачи, провайдер — просто «голосовой движок»**

Любой провайдер подключается снаружи через API/вебхуки:
- VAPI, Retell, ElevenLabs, Ultravox, Bland
- Кастомный LiveKit-агент (самописный, со своими вебхуками)
- n8n/Make pipeline: пришёл звонок → создаётся запись в CallVA → после звонка обогащается данными

Пример flow с ElevenLabs:
1. CallVA триггерит Batch Calling в 11Labs с переменными из своей базы
2. 11Labs звонит → разговор
3. По окончании вебхук → CallVA получает результат, запись, транскрипцию
4. CallVA прогоняет post-call анализ → структурированные данные в запись

**Кастомные поля под задачу (Project-level schema):**
- Каждый проект = своя задача → свой набор полей в записи звонка
- Outbound напоминание о визите: имя, время записи, врач, история визитов
- Outbound реактивация лида: имя, последняя покупка, история покупок, статус лида
- Inbound звонок: телефон, время, транскрипция, запись, extracted structured data
- Один аккаунт → много проектов → каждый со своей схемой

**Post-call анализ:**
- После звонка автоматически извлекаются структурированные данные
- Результат звонка, намерения клиента, следующие шаги
- Может происходить внутри CallVA или через внешний сервис

**Базовый:** баланс → списание за:
- Минуты разговоров

## Amanati

### Суть
Первый пример **вертикального нишевого продукта** на базе CallVA.

### Модель
- White-label CallVA под брендом Amanati
- Клиент получает "панельку Amanati" (под капотом = CallVA)
- Фокус: напоминание о визитах (стоматология и тд)

### Ценообразование (Amanati direct)
- Setup fee: €2,000
- Минуты: ~€0.20–0.30/мин (зависит от объёма)
- Будущее: subscription fee (базовый SLA) + цена минуты в зависимости от объёма

### Технические стеки и себестоимость
- **Gemini Real-Time / NativeAudio**: ~€0.07–0.10/мин — voice-to-voice, быстрее, но дороже, более перебиваемый
- **Классический STT→LLM→TTS**: дешевле, подходит для одноязычных клиник, чуть медленнее
- **Structured data extraction**: после каждого звонка прогон транскрипции через модель (доп. расход токенов)

### Аргументы для продажи
- Быстрее и эффективнее чем нанимать людей
- Нет HR-расходов: поиск, отпуска, больничные, человеческий фактор
- Считаемая экономика — легко сравнить с зарплатой администратора

### Стратегия продаж
Direct outreach по клиникам с конкретным case study.

## Cluster One — зонтичная компания и главное видение

**Написание:** Cluster One (два слова, оба с заглавной). Юрлицо: Clusterone OÜ.
**Синонимы (одно и то же):** Cluster One = AI Cluster = AI Cluster 1 — используются взаимозаменяемо, домены ещё уточняются.
**Утверждённое написание:** AI Cluster One (слово "One", не цифра).
**Альтернатива на рассмотрение:** "AI Cluster 1" (с цифрой) — визуально интересно, выглядит технологично и брендово. Возможно стоит пересмотреть в будущем.

### Миссия
**Обучать людей строить AI-first бизнесы и становиться специалистами по AI-трансформации. Создавать сеть самостоятельных кластеров, каждый из которых несёт AI-first подход дальше.**

### Развёрнутая концепция

**Три уровня ценности Cluster One:**

**1. Строим AI-first бизнесы сами**
Cluster One — не теоретическая школа. Мы сами строим бизнесы по принципу AI-first (CallVA, Amanati и др.). Всё, чему мы учим — мы делаем сами. Это даёт реальные кейсы, реальные инструменты и реальную экспертизу.

**2. Обучаем других строить и перестраивать бизнесы на AI-first**
- Предприниматели приходят, чтобы построить свой бизнес с AI в фундаменте
- Существующие бизнесы приходят, чтобы перестроить процессы: от ручных → к AI-augmented → к AI-first
- Обучение не абстрактное, а практическое: вот инструмент, вот процедура, вот результат

**3. Создаём специалистов по AI-трансформации — новые кластеры**
Это ключевой и самый масштабируемый уровень:
- Человек обучается в Cluster One и становится **специалистом по AI-трансформации**
- Его услуга: помогать бизнесам внедрять AI-first подход — перестраивать процессы, создавать дополнительную ценность через AI
- При этом он **сам ведёт свой бизнес AI-first** — используя те же инструменты и процессы, которым обучился
- Фактически каждый такой специалист = **новый кластер** — автономная единица, которая несёт AI-first подход в свою нишу, свой регион, свою сеть
- Он может работать самостоятельно, но получает от Cluster One: инфраструктуру, инструменты, техподдержку, обновления, комьюнити

**Самовоспроизводящаяся модель:**
```
Cluster One (исходный кластер)
  → обучает специалистов
    → каждый специалист = новый кластер
      → внедряет AI-first в бизнесы своих клиентов
      → сам работает AI-first
      → использует инфраструктуру Cluster One
      → может обучать следующих специалистов
```

**Почему это работает:**
- Система обучает создавать такие же системы — рекурсивная ценность
- Даже если ты занимаешься только консалтингом (помогаешь другим переходить на AI-first) — ты сам должен быть примером этого подхода. И Cluster One даёт тебе для этого всё необходимое
- Cluster One = корневой кластер, точка отсчёта. Он предоставляет: инфраструктуру, системы, обучение, техподдержку — всё, что нужно для успешного функционирования в AI-first мире
- Масштабирование через людей, а не через рекламу: каждый новый кластер привлекает клиентов в своём окружении

### Суть
Cluster One — это **зонтичная компания**, которая строит AI-first будущее. Не просто продукт, а **экосистема**: обучение + инструменты + комьюнити + инфраструктура.

### Бизнес-модель: три типа клиентов

**1. Конечный потребитель (end user)**
- Приходит, учится, получает AI-агента для своего бизнеса
- Пользуется инструментами Cluster One (включая CallVA)

**2. Партнёр/реселлер**
- Учится в Cluster One
- Использует инструменты для себя
- Продаёт и устанавливает клиентам (зарабатывает)
- Получает техническую поддержку и инфраструктуру от Cluster One

**3. Строитель кластера (франшиза)**
- Хочет построить свой кластер / свой бизнес по модели Cluster One
- Реплицирует бизнес-модель: свои процессы AI-first + процессы своих клиентов AI-first
- Использует инфраструктуру и инструменты Cluster One

### Принцип: "Мы продаём то, что используем сами"
- У Cluster One есть свой AI-агент (серия агентов), который работает с учениками и клиентами
- Мы обучаем людей настраивать такие же системы
- Мы даём инфраструктуру, на которой это работает
- Мы разрабатываем инструменты, которые нужны внутри кластера → они же доступны клиентам

### Как стыкуется CallVA
- CallVA = продукт внутри экосистемы Cluster One
- Та же модель: конечный потребитель ИЛИ реселлер
- CallVA дополняется инструментами кластера (ведение лидов, процесс продаж)
- Amanati = вертикальный продукт на CallVA

### Платформа Cluster One: Back-office + Wallet

**Ключевая идея:** У Cluster One должна быть своя платформа (приложение/back-office), чтобы клиент был привязан к экосистеме.

**Back-office включает:**
- Регистрация/логин пользователей
- Управление контентом и контекстом
- Подключение скиллов и инструментов
- Управление AI-агентами
- Дашборд с активностью

**Centralized Wallet:**
- Каждый пользователь имеет wallet, куда заряжает деньги
- Из wallet списываются расходы за разные сервисы:
  - Токены AI-моделей (reselling)
  - CallVA (звонки, минуты)
  - Другие инструменты экосистемы
- Прозрачная детализация: за что и сколько списано
- **Reselling токенов и доступов = мощный источник монетизации**

**Связка с оплатой:**
- Вариант А: Оплата на стороне Cluster One → доступ в Skool группу + платформу
- Вариант Б: Skool чарджит за подписку → Cluster One проверяет активный аккаунт
- В обоих случаях wallet — отдельно для расходников (токены, звонки)

**Как стыкуется с CallVA:**
- Клиент планирует задачу в платформе (напр. прозвонить клиентов)
- Звонки идут через CallVA
- Данные возвращаются в платформу
- Списание из wallet
- Всё в одном месте

## Сессия 2026-03-15 — AI Cluster: концепция Business Brain (AI OS)

*Разговор с KZ на Бали, утро. Идеи и видение зафиксированы дословно.*

### Концепция: Агент-мозг бизнеса

"AI Business Dashboard" — слишком простое название. Речь идёт о принципиально иной концепции.

**Суть:** Выращивается специализированный агент, который становится **живым мозгом / ядром бизнеса**. Он знает о бизнесе **абсолютно всё** — сотрудников, задачи, что выполняется, текущие проблемы, затыки, бэклог, структуру данных и доступы к инструментам. Цель: нет данных, которые этому AI неизвестны.

**Две типа поверхностей взаимодействия:**
1. **Чат / голос** — с агентом можно разговаривать, задавать вопросы о бизнесе, получать ответы голосом
2. **Визуализация** — понятная для человека графическая поверхность (дашборд — один из примеров, не единственный)

**Управление данными:**
- У агента есть тиерированные права доступа: только чтение ИЛИ чтение + обновление
- Данными могут управлять и AI, и люди
- Агент сам создаёт дополнительные surfaces / сервисы поверх этих данных

**Роль AI Cluster:**
- AI Cluster предоставляет готовые строительные блоки (элементы, конфигурации)
- Задача: сделать это доступным для **нетехнических людей**
- Возможно: white-label конфигурации, готовые шаблоны / элементы
- Специалисты AI Cluster строят такие системы для бизнесов

### Варианты названий (сессия 15.03.2026)

| Название | Описание |
|----------|----------|
| **Business Cortex** | Кора головного мозга — управляющий центр. Серьёзно, легко произносится |
| **Nucleus** | Ядро клетки — хранит "генетическую информацию" бизнеса, из него растёт всё |
| **Business Core** | Универсально, понятно сразу |
| **Core** / **Core Agent** | Минималистично. "Дай каждому бизнесу Core" |
| **Entity** | Агент буквально становится бизнес-сущностью. Каждый бизнес имеет свою Entity |
| **AIOS** | AI Operating System — операционная система для бизнеса (сама аббревиатура KZ не очень) |
| **Nucleus OS** | Комбинация Nucleus + OS — звучит круто |
| **Cortex OS** | Комбинация Cortex + OS — тоже сильно |

**Статус названия:** в работе, не финализировано. Направление — что-то вроде `[Brain/Core word] OS`.

### Связь с AI Cluster
Этот продукт / концепция — одна из флагманских демонстраций ценности AI Cluster:
- AI Cluster учит **строить такие системы**
- Предоставляет **инфраструктуру и блоки** для сборки
- Специалисты Cluster One **делают это для бизнесов**
- Агент становится точкой входа для разговора с бизнесом как с единой системой
